# План: Единый пульт управления книгами на маркетплейсах

## Задача
Одна программа для склада, которая выставляет и снимает лоты книг разом на Ozon, Wildberries и Avito. Каталог 50k+ книг, б/у единичные экземпляры (остаток = 1). Работа с нескольких ПК на складе.

## Вводные (согласовано)
- Товар: б/у, единичные экземпляры → остаток каждой книги = 1.
- База: облако/VPS, доступ через браузер с любого склад-ПК.
- API-ключей пока нет → помогаю получить, интеграции подключаю по одной.
- Порядок площадок: поэтапно Ozon → WB → Avito.
- Приоритет: и массовое выставление, и снятие проданного — оба важны.

## Ключевая механика
Книга в одном экземпляре. Продалась на одной площадке → программа **автоматически снимает её с двух других**. Фоновый опрос заказов каждые несколько минут.

**Главный риск (честно):** окно двойной продажи между площадками нельзя убрать в ноль, только сократить частым опросом. На Avito задержка больше — размещение/снятие идёт через XML-фид «Автозагрузки», который Avito забирает по расписанию, а не мгновенно.

## Стек
- Backend: Python + FastAPI
- База: PostgreSQL
- Интерфейс: серверные страницы + HTMX (лёгкие, быстрые на слабых ПК)
- Фоновый планировщик: опрос заказов + генерация фида Avito
- Развёртывание: Docker Compose на VPS

## Модель данных (ядро)
- `books` — единый каталог: внутренний артикул (SKU), название, автор, состояние, цена, фото, описание, статус (в наличии / продана / снята).
- `listings` — привязка книги к площадке: площадка, ID лота на площадке, статус публикации, дата синхронизации.
- `orders` — входящие заказы с площадок (для авто-снятия).
- `sync_log` — журнал всех действий (что, куда, когда, результат) — критично для разбора ошибок на объёме 50k.
- `marketplace_accounts` — хранение ключей/токенов (шифрованно).

## Фазы (после каждой — рабочий результат)

**Фаза 0. Каркас**
Проект, БД, docker-compose, пустой веб-интерфейс, вход по паролю. Запускается на VPS.

**Фаза 1. Каталог + импорт**
- Таблица книг, карточка книги, поиск/фильтры, массовые операции в UI.
- Импорт существующих 50k книг из выгрузок площадок (CSV/XLSX), сопоставление одинаковых книг между площадками в один SKU.
- Пока без обращения к API — проверяем удобство и корректность данных.

**Фаза 2. Ozon (первая живая интеграция)**
- Подключение ключей Ozon Seller API.
- Выставление лота из карточки, обновление цены/остатка.
- Опрос заказов Ozon → при продаже книга помечается «продана».
- Массовое выставление/снятие выбранных книг.

**Фаза 3. Wildberries**
- Аналогично Ozon через WB API.
- Включается кросс-снятие: продажа на Ozon снимает с WB и наоборот.

**Фаза 4. Avito (через фид)**
- Генерация XML-фида «Автозагрузки» из каталога, размещение фида на VPS по постоянной ссылке.
- Настройка расписания забора фидом на стороне Avito.
- Учёт задержки применения в логике снятия.

**Фаза 5. Устойчивость на объёме**
- Пакетная обработка и лимиты запросов площадок (rate limits).
- Повторные попытки при сбоях, разбор ошибок в UI.
- Бэкапы базы, мониторинг фонового опроса.

## Что нужно от тебя по ходу
- Ключи API: инструкции дам в начале Фазы 2 (Ozon), затем WB, затем Avito.
- Примеры выгрузок каталога с площадок (CSV/XLSX) для Фазы 1 — чтобы правильно сопоставить книги.

## Начинаю с Фазы 0 + Фазы 1
Сразу увидишь каталог с импортированными книгами в браузере, ещё до подключения площадок.
